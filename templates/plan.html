Aarogyam ScanAI: Detailed Project Plan (Independent Phases)

This plan divides the project into six self-contained modules. Each module can be developed and tested independently before final integration, allowing for parallel work and clear progress tracking.

📌 Phase 1: Chest X-Ray AI Module Development

Goal: To develop a complete, standalone AI model that can analyze chest X-ray images for various lung diseases.

Team Roles:

  • Ankit (Data Acquisition & Preprocessing Lead): Identify, download, and organize large datasets like NIH Chest X-ray and RSNA Pneumonia Detection Challenge. Handle DICOM, PNG, JPG files using `pydicom` and `PIL`.
  • Gautam (Preprocessing Specialist): Implement preprocessing techniques like resizing images to 224x224 pixels, converting to grayscale, noise reduction, and data augmentation. Split data into training, validation, and testing sets.
  • Rahul (Model Development Lead): Design and implement a suitable CNN architecture (e.g., ResNet-101) for chest X-ray analysis. Train the model and perform hyperparameter tuning.
  • Prince (Prediction & Evaluation Specialist): Use the trained model to make disease predictions. Evaluate the model's performance using metrics like Accuracy, ROC-AUC, Sensitivity, and Specificity. Save the model for reuse (in HDF5 or SavedModel format).

Detailed Steps:

  1. Identify and acquire necessary datasets (NIH Chest X-ray, RSNA Pneumonia).
  2. Convert various image formats (DICOM, PNG, JPG) to a standard format.
  3. Develop the image preprocessing pipeline (resizing, normalization, noise reduction).
  4. Expand the dataset using data augmentation techniques.
  5. Select and implement the CNN model architecture.
  6. Train the model for multi-class classification (Normal, Pneumonia, TB, Pneumothorax, Cardiomegaly, Lung Cancer nodules).
  7. Rigorously evaluate and improve the model's performance.
  8. Save the final model in a portable format.

Outcome: A standalone Python module containing a trained AI model (.h5 or .pb file) and a function that takes a chest X-ray image path as input and returns the disease prediction (label, confidence score) as a structured dictionary. This module can function entirely on its own.

Key Technologies & Tools: Python, NumPy, Pandas, pydicom, PIL, OpenCV, TensorFlow/Keras, PyTorch, CNN, ResNet-101.

📌 Phase 2: Brain CT/MRI AI Module Development

Goal: To create a separate, specialized AI model to detect brain conditions (e.g., hemorrhage, stroke, tumor) from CT and MRI scans.

Team Roles:

  • Ankit (Data Handling Lead): Find and organize brain imaging datasets like LIDC-IDRI, BraTS, and ISLES. Use `PyDicom` and `NiBabel` to handle DICOM/NIfTI files and convert them to a suitable format for the AI model.
  • Gautam (Preprocessing Specialist): Implement special preprocessing techniques for brain scans (e.g., slice normalization, skull stripping, 3D volume handling).
  • Rahul (Model Development Lead): Design and implement 2D/3D CNN architectures (e.g., 3D ResNet, U-Net). Train and optimize the model.
  • Prince (Prediction & Evaluation Specialist): Use the trained model to predict brain conditions. Evaluate the model for both classification (e.g., hemorrhage, stroke) and segmentation (e.g., tumor).

Detailed Steps:

  1. Identify and acquire brain imaging datasets (RSNA, ISLES, BraTS).
  2. Prepare DICOM/NIfTI data for preprocessing and model input.
  3. Develop a preprocessing pipeline specific to brain scans.
  4. Select and implement 2D and 3D CNN model architectures.
  5. Train the model to detect various brain conditions (e.g., Hemorrhage, Ischemic Stroke, Atrophy, Brain Tumor lesion).
  6. Evaluate model performance using classification and segmentation metrics (e.g., Dice Coefficient).
  7. Save the final model.

Outcome: A standalone Python module containing a trained AI model and a function that takes a brain scan image path as input and returns the prediction (condition, severity, segmentation mask if applicable) as a structured dictionary. This module will function independently of the chest X-ray module.

Key Technologies & Tools: Python, PyDicom, NiBabel, OpenCV, PyTorch, MONAI, CNN, 2D/3D ResNet, U-Net.

📌 Phase 3: Core UI & Frontend Dashboard Development

Goal: To build a fully functional and appealing user interface (UI) for the web application, including a home page, upload form, history page, login, and registration.

Team Roles:

  • Prince (UI/UX Design & HTML/CSS Lead): Design wireframes and mockups for all web pages (Home, About, Plan, Upload, History, Login, Register). Develop the HTML structure and CSS styling (all related `.css` files), ensuring a responsive and user-friendly design.
  • Rahul (Frontend JavaScript & Interactivity): Write client-side JavaScript to handle user interactions like menu toggling, basic form validation, and placeholders for dynamic content display. Set up all navigation links (`url_for`) to the correct Flask routes.

Detailed Steps:

  1. Create the UI/UX design for all main pages of the website.
  2. Develop all HTML templates, including `base.html`.
  3. Create dedicated CSS files for each page (e.g., `style.css`, `about.css`, `plan.css`, `login.css`, `register.css`, `history.css`) and link them.
  4. Implement JavaScript for mobile menu toggling and basic client-side validation on form inputs.
  5. Properly configure all navigation links using Flask's `url_for` function.

Outcome: A set of static HTML/CSS/JS files representing the complete visual and interactive frontend. This module will be visually complete and handle basic user interactions but will not yet be connected to a backend or AI model.

Key Technologies & Tools: HTML5, CSS3, JavaScript (Vanilla JS), Bootstrap (if used), Jinja2 templating.

📌 Phase 4: Backend & Database System

Goal: To develop the core server-side logic for user authentication, session management, and secure data storage.

Team Roles:

  • Ankit (Database Design & Implementation Lead): Set up the SQLite database (`user_data.db`). Design and implement the detailed database schema for tables like `users` and `history`, ensuring data integrity and relationships.
  • Rahul (Flask Routes & API Development): Establish the Flask application structure. Create all necessary API endpoints for user registration, login, logout, and other required data interactions.
  • Prince (User Authentication & Session Management): Use Flask's `session` object to securely manage user sessions. Implement password hashing using a library like `werkzeug.security` to store passwords securely.

Detailed Steps:

  1. Initial setup and configuration of the Flask application.
  2. Create utility functions for SQLite database connections (`get_db`, `close_connection`).
  3. Develop the database initialization script to create the `users` and `history` tables.
  4. Implement the Flask routes and logic for user registration (`/register`) and login (`/login`).
  5. Implement session management for login status and user-specific data.
  6. Define the API endpoints to receive and store data from the future AI and report modules.

Outcome: A fully functional Flask server that handles user registration, login/logout, and creates and accesses database tables. This module will be ready to receive and securely store data from other modules.

Key Technologies & Tools: Python, Flask, SQLite3, `werkzeug.security`, `g` (Flask context globals).

📌 Phase 5: Report Generation & Visualization Service

Goal: To create a standalone service for generating diagnostic reports and visual heatmaps based on AI predictions.

Team Roles:

  • Gautam (Heatmap Visualization Lead): Implement the Grad-CAM (Gradient-weighted Class Activation Mapping) technique. This will automatically highlight affected regions on a scan, providing visual explanation for the AI's prediction.
  • Ankit (Report Generation Specialist): Use a library like `ReportLab` or `FPDF` to create professional, downloadable PDF reports that include the AI predictions and the generated heatmap. The report will also contain structured textual diagnostic suggestions.

Detailed Steps:

  1. Implement the Grad-CAM algorithm which takes an image and model output as input.
  2. Develop the functionality to overlay the generated heatmap onto the original scan image.
  3. Design a PDF report template that includes the AI results, heatmap image, upload date, and other relevant information.
  4. Add logic to make the report multilingual (e.g., using Google Translate API or with pre-translated strings).
  5. Ensure the report is clear, concise, and easy to understand.

Outcome: A standalone Python module containing functions that take AI prediction data and an original image as input, and output a heatmap image file (.png) and a downloadable PDF report file (.pdf). This module can be developed and tested with dummy data without waiting for the completion of the other AI modules.

Key Technologies & Tools: Python, OpenCV, Matplotlib, ReportLab/FPDF, Grad-CAM libraries (e.g., `tf-keras-vis` or custom implementation).

📌 Phase 6: Full System Integration & Deployment

Goal: To combine all independent modules into a cohesive, fully functional application and prepare it for deployment.

Team Roles:

  • Ankit (Backend Integration Lead): Link the UI (Phase 3), database (Phase 4), and all AI models (Phase 1 & 2) together. Ensure the seamless flow of data and correct functioning of API calls.
  • Rahul (Backend Routing & Optimization): Implement the logic to route the image to the correct AI model (chest X-ray or brain scan) based on the uploaded file type. Implement error handling and logging throughout the system.
  • Prince (Frontend Integration & User Experience): Update the frontend (Phase 3) to display the AI predictions, heatmaps, and reports on the user dashboard. Add report download links and ensure a cohesive user experience.
  • Gautam (Testing & Quality Assurance): Perform end-to-end testing of the entire system. Identify and fix bugs. Assist with performance optimization.

Detailed Steps:

  1. Integrate the Chest X-ray and Brain AI modules into the Flask backend.
  2. Develop logic to automatically detect the type of uploaded image and send it to the correct AI module.
  3. Update the frontend to display the AI results, heatmap images, and report download links.
  4. Connect the report generation and visualization module to the backend.
  5. Implement comprehensive error handling and logging across the system.
  6. Conduct extensive testing (unit, integration, end-to-end).
  7. Prepare the application for production deployment (e.g., Dockerization, Gunicorn/Nginx setup).
  8. Finalize project documentation and presentation.

Outcome: A complete, integrated web application, Aarogyam ScanAI, ready for real-world use and scalable for future enhancements.

Key Technologies & Tools: All technologies from previous phases, plus potentially Docker, Gunicorn, Nginx (for production deployment).

📌 Phase 1: Chest X-Ray AI Module Development

लक्ष्य: एक पूर्ण, स्टैंडअलोन AI मॉडल विकसित करना जो विभिन्न फेफड़ों की बीमारियों के लिए चेस्ट एक्स-रे छवियों का विश्लेषण कर सके।

टीम भूमिकाएँ:

  • अंकित (Data Acquisition & Preprocessing Lead): NIH चेस्ट एक्स-रे और RSNA निमोनिया डिटेक्शन चैलेंज जैसे बड़े डेटासेट को खोजना, डाउनलोड करना और व्यवस्थित करना। DICOM, PNG, JPG फ़ाइलों को `pydicom` और `PIL` का उपयोग करके संभालना।
  • गौतम (Preprocessing Specialist): छवियों का आकार 224x224 पिक्सेल में बदलना, ग्रेस्केल रूपांतरण, शोर हटाना (noise reduction), और डेटा ऑग्मेंटेशन (augmentation) जैसी प्रीप्रोसेसिंग तकनीकों को लागू करना। डेटा को ट्रेनिंग, वैलिडेशन और टेस्टिंग सेट में विभाजित करना।
  • राहुल (Model Development Lead): चेस्ट एक्स-रे विश्लेषण के लिए उपयुक्त CNN आर्किटेक्चर (जैसे ResNet-101) का डिज़ाइन और कार्यान्वयन। मॉडल को प्रशिक्षित करना और हाइपरपैरामीटर ट्यूनिंग करना।
  • प्रिंस (Prediction & Evaluation Specialist): प्रशिक्षित मॉडल का उपयोग करके बीमारी की भविष्यवाणी करना। मॉडल के प्रदर्शन का मूल्यांकन करना (Accuracy, ROC-AUC, Sensitivity, Specificity)। मॉडल को पुनः उपयोग के लिए सेव करना (HDF5 या SavedModel फॉर्मेट में)।

विस्तृत चरण:

  1. आवश्यक डेटासेट (NIH Chest X-ray, RSNA Pneumonia) की पहचान और अधिग्रहण।
  2. DICOM, PNG, JPG जैसे विभिन्न इमेज फॉर्मेट्स को मानक फॉर्मेट में परिवर्तित करना।
  3. इमेज प्रीप्रोसेसिंग पाइपलाइन (resizing, normalization, noise reduction) का विकास।
  4. डेटा ऑग्मेंटेशन तकनीकों का उपयोग करके डेटासेट का विस्तार करना।
  5. CNN मॉडल आर्किटेक्चर का चयन और कार्यान्वयन।
  6. मल्टी-क्लास क्लासिफिकेशन के लिए मॉडल को प्रशिक्षित करना (Normal, Pneumonia, TB, Pneumothorax, Cardiomegaly, Lung Cancer nodules)।
  7. मॉडल के प्रदर्शन का कठोरता से मूल्यांकन करना और सुधार करना।
  8. अंतिम मॉडल को एक पोर्टेबल फॉर्मेट में सेव करना।

परिणाम: एक स्टैंडअलोन पायथन मॉड्यूल जिसमें एक प्रशिक्षित AI मॉडल (.h5 या .pb फ़ाइल) और एक फ़ंक्शन शामिल है जो एक चेस्ट एक्स-रे इमेज पाथ को इनपुट के रूप में लेता है और बीमारी की भविष्यवाणी (डिजीज लेबल, आत्मविश्वास स्कोर) को एक संरचित डिक्शनरी के रूप में देता है। यह मॉड्यूल पूरी तरह से अपने आप काम कर सकता है।

मुख्य तकनीकें और उपकरण: Python, NumPy, Pandas, pydicom, PIL, OpenCV, TensorFlow/Keras, PyTorch, CNN, ResNet-101.

📌 Phase 2: Brain CT/MRI AI Module Development

लक्ष्य: CT और MRI स्कैन से मस्तिष्क की स्थितियों (जैसे रक्तस्राव, स्ट्रोक, ट्यूमर) का पता लगाने के लिए एक अलग, विशेष AI मॉडल बनाना।

टीम भूमिकाएँ:

  • अंकित (Data Handling Lead): LIDC-IDRI, BraTS, ISLES जैसे मस्तिष्क इमेजिंग डेटासेट को खोजना और व्यवस्थित करना। DICOM/NIfTI फ़ाइलों को संभालने और उन्हें AI मॉडल के लिए उपयुक्त फॉर्मेट में बदलने के लिए `PyDicom` और `NiBabel` का उपयोग करना।
  • गौतम (Preprocessing Specialist): मस्तिष्क स्कैन के लिए विशेष प्रीप्रोसेसिंग तकनीकों (जैसे स्लाइस नॉर्मलाइज़ेशन, स्कल स्ट्रिपिंग, 3D वॉल्यूम हैंडलिंग) को लागू करना।
  • राहुल (Model Development Lead): 2D/3D CNN आर्किटेक्चर (जैसे 3D ResNet, U-Net) का डिज़ाइन और कार्यान्वयन। मॉडल को प्रशिक्षित करना और ऑप्टिमाइज़ करना।
  • प्रिंस (Prediction & Evaluation Specialist): प्रशिक्षित मॉडल का उपयोग करके मस्तिष्क की स्थितियों की भविष्यवाणी करना। क्लासिफिकेशन (जैसे रक्तस्राव, स्ट्रोक) और सेगमेंटेशन (जैसे ट्यूमर) दोनों के लिए मॉडल का मूल्यांकन करना।

विस्तृत चरण:

  1. मस्तिष्क इमेजिंग डेटासेट (RSNA, ISLES, BraTS) की पहचान और अधिग्रहण।
  2. DICOM/NIfTI डेटा को प्रीप्रोसेसिंग और मॉडल इनपुट के लिए तैयार करना।
  3. मस्तिष्क स्कैन के लिए विशिष्ट प्रीप्रोसेसिंग पाइपलाइन का विकास।
  4. 2D और 3D CNN मॉडल आर्किटेक्चर का चयन और कार्यान्वयन।
  5. मॉडल को विभिन्न मस्तिष्क स्थितियों (जैसे Hemorrhage, Ischemic Stroke, Atrophy, Brain Tumor lesion) का पता लगाने के लिए प्रशिक्षित करना।
  6. क्लासिफिकेशन और सेगमेंटेशन मेट्रिक्स (जैसे Dice Coefficient) का उपयोग करके मॉडल के प्रदर्शन का मूल्यांकन करना।
  7. अंतिम मॉडल को सेव करना।

परिणाम: एक स्टैंडअलोन पायथन मॉड्यूल जिसमें एक प्रशिक्षित AI मॉडल और एक फ़ंक्शन शामिल है जो एक मस्तिष्क स्कैन इमेज पाथ को इनपुट के रूप में लेता है और भविष्यवाणी (स्थिति, गंभीरता, यदि लागू हो तो सेगमेंटेशन मास्क) को एक संरचित डिक्शनरी के रूप में देता है। यह मॉड्यूल चेस्ट एक्स-रे मॉड्यूल से स्वतंत्र रूप से कार्य करेगा।

मुख्य तकनीकें और उपकरण: Python, PyDicom, NiBabel, OpenCV, PyTorch, MONAI, CNN, 2D/3D ResNet, U-Net.

📌 Phase 3: Core UI & Frontend Dashboard Development

लक्ष्य: वेब एप्लिकेशन के लिए एक पूरी तरह से कार्यात्मक और आकर्षक यूजर इंटरफेस (UI) बनाना, जिसमें होम पेज, अपलोड फ़ॉर्म, हिस्ट्री पेज, लॉगिन और पंजीकरण शामिल हों।

टीम भूमिकाएँ:

  • प्रिंस (UI/UX Design & HTML/CSS Lead): सभी वेब पेजों (Home, About, Plan, Upload, History, Login, Register) के लिए वायरफ्रेम और मॉकअप डिज़ाइन करना। एक प्रतिक्रियाशील (responsive) और उपयोगकर्ता के अनुकूल डिज़ाइन सुनिश्चित करते हुए HTML संरचना और CSS स्टाइलिंग (सभी संबंधित `.css` फ़ाइलें) विकसित करना।
  • राहुल (Frontend JavaScript & Interactivity): क्लाइंट-साइड जावास्क्रिप्ट लिखना जो मेनू टॉगलिंग, बुनियादी फ़ॉर्म सत्यापन (validation), और गतिशील सामग्री प्रदर्शन के लिए प्लेसहोल्डर जैसी उपयोगकर्ता इंटरैक्शन को संभालता है। सभी नेविगेशन लिंक (`url_for`) को सही Flask रूट्स पर सेट करना।

विस्तृत चरण:

  1. वेबसाइट के सभी मुख्य पेजों के लिए UI/UX डिज़ाइन बनाना।
  2. `base.html` सहित सभी HTML टेम्पलेट्स को विकसित करना।
  3. प्रत्येक पेज के लिए समर्पित CSS फ़ाइलें (जैसे `style.css`, `about.css`, `plan.css`, `login.css`, `register.css`, `history.css`) बनाना और उन्हें लिंक करना।
  4. मोबाइल मेनू टॉगलिंग, फ़ॉर्म इनपुट पर बुनियादी क्लाइंट-साइड सत्यापन के लिए जावास्क्रिप्ट लागू करना।
  5. सभी नेविगेशन लिंक्स को Flask के `url_for` फंक्शन का उपयोग करके सही ढंग से कॉन्फ़िगर करना।

परिणाम: स्थिर HTML/CSS/JS फ़ाइलों का एक सेट जो संपूर्ण दृश्य और इंटरैक्टिव फ्रंटएंड का प्रतिनिधित्व करता है। यह मॉड्यूल नेत्रहीन रूप से पूर्ण होगा और बुनियादी उपयोगकर्ता इंटरैक्शन को संभालेगा, लेकिन अभी तक बैकएंड या AI मॉडल से जुड़ा नहीं होगा।

मुख्य तकनीकें और उपकरण: HTML5, CSS3, JavaScript (Vanilla JS), Bootstrap (यदि उपयोग किया गया हो), Jinja2 templating.

📌 Phase 4: Backend & Database System

लक्ष्य: उपयोगकर्ता प्रमाणीकरण, सत्र प्रबंधन और सुरक्षित डेटा भंडारण के लिए कोर सर्वर-साइड लॉजिक विकसित करना।

टीम भूमिकाएँ:

  • अंकित (Database Design & Implementation Lead): SQLite डेटाबेस (`user_data.db`) स्थापित करना। `users` और `history` जैसी तालिकाओं के लिए विस्तृत डेटाबेस स्कीमा (schema) को डिज़ाइन और लागू करना, डेटा अखंडता और संबंध (relationships) सुनिश्चित करना।
  • राहुल (Flask Routes & API Development): Flask एप्लिकेशन संरचना स्थापित करना। उपयोगकर्ता पंजीकरण, लॉगिन, लॉगआउट और अन्य आवश्यक डेटा इंटरैक्शन के लिए सभी आवश्यक API एंडपॉइंट बनाना।
  • प्रिंस (User Authentication & Session Management): उपयोगकर्ता सत्रों को सुरक्षित रूप से प्रबंधित करने के लिए Flask के सत्र (`session`) ऑब्जेक्ट का उपयोग करना। पासवर्ड को सुरक्षित रूप से स्टोर करने के लिए `werkzeug.security` जैसी लाइब्रेरी का उपयोग करके पासवर्ड हैशिंग लागू करना।

विस्तृत चरण:

  1. Flask एप्लिकेशन का प्रारंभिक सेटअप और कॉन्फ़िगरेशन।
  2. SQLite डेटाबेस कनेक्शन (`get_db`, `close_connection`) के लिए यूटिलिटी फ़ंक्शंस बनाना।
  3. `users` और `history` तालिकाओं को बनाने के लिए डेटाबेस इनिशियलाइज़ेशन स्क्रिप्ट विकसित करना।
  4. उपयोगकर्ता पंजीकरण (`/register`) और लॉगिन (`/login`) के लिए Flask रूट्स और लॉजिक लागू करना।
  5. लॉगिन स्थिति और उपयोगकर्ता-विशिष्ट डेटा के लिए सत्र प्रबंधन लागू करना।
  6. भविष्य के AI और रिपोर्ट मॉड्यूल से डेटा प्राप्त करने और स्टोर करने के लिए API एंडपॉइंट्स को परिभाषित करना।

परिणाम: एक पूरी तरह से कार्यात्मक Flask सर्वर जो उपयोगकर्ता पंजीकरण, लॉगिन/लॉगआउट को संभालता है, और डेटाबेस तालिकाओं को बनाता है और एक्सेस करता है। यह मॉड्यूल अन्य मॉड्यूलों से डेटा प्राप्त करने और उसे सुरक्षित रूप से स्टोर करने के लिए तैयार होगा।

मुख्य तकनीकें और उपकरण: Python, Flask, SQLite3, `werkzeug.security`, `g` (Flask context globals).

📌 Phase 5: Report Generation & Visualization Service

लक्ष्य: AI भविष्यवाणियों के आधार पर डायग्नोस्टिक रिपोर्ट और विज़ुअल हीटमैप उत्पन्न करने के लिए एक स्टैंडअलोन सेवा बनाना।

टीम भूमिकाएँ:

  • गौतम (Heatmap Visualization Lead): Grad-CAM (Gradient-weighted Class Activation Mapping) तकनीक को लागू करना। यह स्कैन पर प्रभावित क्षेत्रों को स्वचालित रूप से हाइलाइट करेगा, जिससे AI की भविष्यवाणी के लिए दृश्य स्पष्टीकरण मिलेगा।
  • अंकित (Report Generation Specialist): AI भविष्यवाणियों और उत्पन्न हीटमैप को शामिल करते हुए पेशेवर, डाउनलोड करने योग्य PDF रिपोर्ट बनाने के लिए `ReportLab` या `FPDF` जैसी लाइब्रेरी का उपयोग करना। रिपोर्ट में संरचित पाठ्य निदान सुझाव भी शामिल होंगे।

विस्तृत चरण:

  1. Grad-CAM एल्गोरिथम का कार्यान्वयन जो एक इमेज और मॉडल आउटपुट को इनपुट के रूप में लेता है।
  2. उत्पन्न हीटमैप को मूल स्कैन इमेज पर ओवरले करने की कार्यक्षमता विकसित करना।
  3. PDF रिपोर्ट टेम्पलेट का डिज़ाइन जिसमें AI परिणाम, हीटमैप इमेज, अपलोड की तारीख और अन्य प्रासंगिक जानकारी शामिल हो।
  4. रिपोर्ट को बहुभाषी (multilingual) बनाने के लिए लॉजिक जोड़ना (जैसे Google Translate API का उपयोग करके या पूर्व-अनुवादित स्ट्रिंग्स के साथ)।
  5. यह सुनिश्चित करना कि रिपोर्ट स्पष्ट, संक्षिप्त और समझने में आसान हो।

परिणाम: एक स्टैंडअलोन पायथन मॉड्यूल जिसमें फ़ंक्शंस शामिल हैं जो AI भविष्यवाणी डेटा और एक मूल इमेज को इनपुट के रूप में लेते हैं, और एक हीटमैप इमेज फ़ाइल (.png) और एक डाउनलोड करने योग्य PDF रिपोर्ट फ़ाइल (.pdf) आउटपुट करते हैं। यह मॉड्यूल अन्य AI मॉड्यूलों के पूरा होने का इंतजार किए बिना डमी डेटा के साथ विकसित और परीक्षण किया जा सकता है।

मुख्य तकनीकें और उपकरण: Python, OpenCV, Matplotlib, ReportLab/FPDF, Grad-CAM libraries (e.g., `tf-keras-vis` या कस्टम कार्यान्वयन).

📌 Phase 6: Full System Integration & Deployment

लक्ष्य: सभी स्वतंत्र मॉड्यूलों को एक सुसंगत, पूरी तरह कार्यात्मक एप्लिकेशन में जोड़ना और इसे डिप्लॉयमेंट के लिए तैयार करना।

टीम भूमिकाएँ:

  • अंकित (Backend Integration Lead): UI (Phase 3), डेटाबेस (Phase 4), और सभी AI मॉडल (Phase 1 & 2) को एक साथ लिंक करना। डेटा के निर्बाध प्रवाह और API कॉल्स के सही संचालन को सुनिश्चित करना।
  • राहुल (Backend Routing & Optimization): अपलोड की गई फ़ाइल के प्रकार के आधार पर सही AI मॉडल (चेस्ट एक्स-रे या ब्रेन स्कैन) पर इमेज को रूट करने के लिए लॉजिक लागू करना। पूरे सिस्टम में त्रुटि हैंडलिंग और लॉगिंग को लागू करना।
  • प्रिंस (Frontend Integration & User Experience): AI भविष्यवाणियों, हीटमैप और रिपोर्ट को उपयोगकर्ता डैशबोर्ड पर प्रदर्शित करने के लिए फ्रंटएंड (Phase 3) को अपडेट करना। रिपोर्ट डाउनलोड लिंक्स को जोड़ना और एक सुसंगत उपयोगकर्ता अनुभव सुनिश्चित करना।
  • गौतम (Testing & Quality Assurance): पूरे सिस्टम का एंड-टू-एंड परीक्षण करना। बग्स की पहचान करना और उन्हें ठीक करना। प्रदर्शन अनुकूलन (performance optimization) में सहायता करना।

विस्तृत चरण:

  1. Flask बैकएंड में चेस्ट एक्स-रे और ब्रेन AI मॉड्यूलों को एकीकृत करना।
  2. अपलोड की गई इमेज के प्रकार का स्वचालित रूप से पता लगाने और उसे सही AI मॉड्यूल पर भेजने के लिए लॉजिक विकसित करना।
  3. फ्रंटएंड को अपडेट करना ताकि AI परिणाम, हीटमैप इमेज और रिपोर्ट डाउनलोड लिंक्स प्रदर्शित हो सकें।
  4. रिपोर्ट जनरेशन और विज़ुअलाइज़ेशन मॉड्यूल को बैकएंड से जोड़ना।
  5. पूरे सिस्टम में व्यापक त्रुटि हैंडलिंग और लॉगिंग लागू करना।
  6. व्यापक परीक्षण (यूनिट, इंटीग्रेशन, एंड-टू-एंड) करना।
  7. उत्पादन डिप्लॉयमेंट (जैसे Dockerization, Gunicorn/Nginx सेटअप) के लिए एप्लिकेशन तैयार करना।
  8. अंतिम प्रोजेक्ट डॉक्यूमेंटेशन और प्रेजेंटेशन तैयार करना।

परिणाम: एक पूर्ण, एकीकृत वेब एप्लिकेशन, Aarogyam ScanAI, जो वास्तविक दुनिया के उपयोग के लिए तैयार है और भविष्य के संवर्द्धन के लिए स्केलेबल है।

मुख्य तकनीकें और उपकरण: सभी पिछले चरणों की तकनीकें, साथ ही संभावित रूप से Docker, Gunicorn, Nginx (उत्पादन डिप्लॉयमेंट के लिए).
